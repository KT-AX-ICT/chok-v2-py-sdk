# ADR-004 — 리플레이어 데이터 배치 및 원천 경로

- 상태: 제안
- 날짜: 2026-07-15

## 맥락

데모 시스템은 SN 전체 데이터셋을 타임시프트 재생해 "실서비스 측 원천 로그"를 만들어낸다.
성격이 다른 두 데이터 위치가 있는데, 둘 다 정규화 이전이라 `rawdata` 같은 이름으로는 명명하지 않으며, 아래 경로와 역할 구분을 명시한다.

- **전체 데이터셋** — 리플레이어가 **읽는** 전체 기간 원본. 불변 아카이브.
- **원천 로그 데이터** — 리플레이어가 타임스탬프를 현재 시각으로 치환해 **쓰는** 곳. SDK tailer 의
  관측 대상. 실행마다 자라는 런타임 산출물.

또한 `source_root` 기본값이 `./sample_data` 였는데, 실제 폴더는 `examples/sample_data` 라 저장소
루트에서 실행해도 존재하지 않는 경로를 가리켰다.

## 결정

### 경로

| 경로 | 설정 키 | 역할 | git |
|---|---|---|---|
| `datasets/sn/` | `RCA_DATASET_ROOT` | 리플레이어가 읽는 전체 기간 원본 | 일부만 커밋 (아래) |
| `var/` | `RCA_SOURCE_ROOT` | 리플레이어 출력 = tailer 관측 대상 | 커밋 안 함 |

`var/` 는 리플레이어가 실행 시 만들어내는 런타임 산출물이라 커밋하지 않는다. 재생 결과는 재현 가능하고,
매 실행마다 타임스탬프가 바뀌어 diff 가 무의미하다.

### 데이터셋 커밋 범위

`datasets/sn` 전체는 **833MB / 9,009 파일**이다. 그대로 커밋하면 히스토리에 영구히 남고 클론이 무거워진다.
git 은 diff 가 의미 없는 대용량 데이터에 맞지 않는다. 그렇다고 전부 gitignore 하면 클론만으로는 데모가
돌지 않는다.

**MVP 3종 시나리오의 log/metric/trace 만 커밋한다** (167MB / 95 파일). 기준선은 "클론만으로 데모 3종이
돈다"이다.

| 대상 | 용량 | 커밋 | 근거 |
|---|---|---|---|
| MVP 3종 × log/metric/trace | 167MB | O | 리플레이어 입력. 데모의 최소 구성 |
| 나머지 10종 | ~530MB | X | 별도 공유. MVP 범위 밖 |
| `coverage_data/` | 135MB | X | ADR-003 에서 실시간 파이프라인 제외 결정 |
| `api_responses/` | 1.4MB | X | 리플레이어가 읽지 않음 |

MVP 3종 접두어: `Perf_CPU_Contention_*`, `Svc_Kill_Media_*`, `Code_Stop_MediaService_*`.

제외된 모달리티 폴더도 **README 를 두어 디렉터리 구조는 git 에 유지한다.** 받아온 데이터를 어디에 풀지가
구조로 드러나고, 각 폴더가 왜 비어 있는지가 그 자리에서 설명된다.

제외 규칙은 저장소 루트 `.gitignore` 에 둔다. 시나리오 추가는 `!datasets/sn/<모달리티>/<접두어>_*/` 한 줄.

원천 로그를 만드는 경로는 리플레이어 하나로 통일한다. 발췌 fixture(`examples/sample_data/`)를 두고
리플레이어 없이 도는 두 번째 경로는 만들지 않는다 — 유지 비용에 비해 리플레이어를 돌리는 것과 크게
다르지 않고, 두 경로가 갈리면 tailer 가 보는 데이터 형태도 갈린다.

### 경로 해석 기준 — CWD

두 경로 모두 저장소 기준 상대경로를 기본값으로 둔다 (`./var`, `./datasets/sn`).

**상대경로는 `.env` 위치도 `config.py` 위치도 아닌, 프로세스의 작업 디렉터리(CWD) 기준으로 풀린다.**
`config.py` 의 `env_file=".env"` 역시 CWD 기준이라, CWD 가 어긋나면 `.env` 파일 자체를 못 찾고 기본값으로
조용히 떨어진다. 의존이 이중이다. 실측:

```
저장소 루트에서:  .env 읽힘   → ./var → C:\chok-v2-py-sdk\var
scripts/ 에서:    .env 못 찾음 → 기본값 → C:\var  (존재하지 않음)
```

따라서 **SDK 는 항상 저장소 루트에서 실행한다.** 이 전제가 깨지면 아래 "디렉터리 부재 체크"가 잡는다.

저장소 루트를 `__file__` 기준으로 계산하는 방법(`Path(__file__).parents[2]`)은 채택하지 않는다. SDK 가
설치되면 `site-packages` 밑에서 엉뚱한 경로를 짚고, 배포 형상엔 저장소 루트라는 개념 자체가 없다.

### 디렉터리 부재 체크

tailer 는 없는 경로를 봐도 예외 없이 이벤트 0건을 낸다. 트리거가 안 걸리니 **설정 오류가 "이상 없음"과
구분되지 않는다.** 에러가 아니라 정상으로 보이는 것이 문제다. 그래서 둘을 나눈다:

| 상황 | 판정 | 동작 |
|---|---|---|
| `source_root` 또는 모달리티 하위 디렉터리 부재 | 설정 오류 | 기동 시 실패 — 해석된 절대경로와 CWD 를 함께 알린다 |
| 디렉터리는 있으나 파일 없음 / 신규 라인 없음 | 정상 | 0건 |

경로는 기동 시 1회 검증한다. 30초 루프 안에서 매번 확인하지 않는다 — 설정은 실행 중 바뀌지 않는다.
실패 메시지에 CWD 를 넣는 이유는, 이 오류의 실제 원인이 대개 "경로가 틀렸다"가 아니라 "엉뚱한 데서
실행했다"이기 때문이다.

### 모달리티 레이아웃

콜렉터는 모달리티별로 `source_root` 하위 디렉터리 하나씩을 소유하고, 파일명이 `service` 가 된다.

```
<source_root>/
├── log/media.jsonl
├── metric/media.jsonl
└── trace/media.jsonl
```

`source_root` 는 세 모달리티 디렉터리의 **부모**다. `var/log/<service>.jsonl` 은 리눅스 FHS 의
`/var/log/<service>` 와 대응한다. metric/trace 가 형제로 붙는 것은 데모의 편의(실제 프로덕션에선 각각
scrape/OTLP 로 들어오며 디스크에 남지 않는다)이며, 그 전환은 `Collector.poll()` 구현 교체로 흡수한다.

포맷은 세 모달리티 모두 JSONL. tail 이 "새로 생긴 줄 읽기"라 줄 단위여야 하고, 포맷이 같으면 파싱
분기가 없다. `raw_ref` 에 `log/media.jsonl:1423` 형태로 원본을 참조한다.

### 시나리오 축

시나리오(cpu / kill_media / code_media) 선택은 **리플레이어 인자**로 받는다. 경로에 인코딩하지 않는다.
어느 시나리오를 재생하든 출력은 같은 `var/` 이므로 `source_root` 는 고정이고, SDK 는 지금 무엇이
재생 중인지 알 필요가 없다 — tail 시점의 관심사가 아니다.

### 리플레이어 CLI

`var/` 를 **비우고 다시 적재하는 옵션**을 제공한다. 재생은 append 라서, 같은 시나리오를 다시 돌리면
이전 실행 잔여물 위에 쌓여 tailer 가 중복 이벤트를 본다.

## 결과/영향

- `config.py` — `source_root` 기본값 `./sample_data` → `./var`, `dataset_root` 신설(`./datasets/sn`).
- **Runner 에 기동 시 경로 검증이 필요하다** (위 "디렉터리 부재 체크"). 이것이 없으면 CWD 전제가
  깨졌을 때 조용히 0건으로 흘러간다. tailer 구현보다 먼저 들어가야 한다.
- `scripts/generate_sample_data.py` 와 `examples/sample_data/` 를 폐기하고 리플레이어로 대체한다
  (발췌 로직이 `TODO` 인 스캐폴드라 잃을 것이 없음). 시나리오 → SN 디렉터리 매핑만 승계한다:
  `cpu=Perf_CPU_Contention`, `kill_media=Svc_Kill_Media`, `code_media=Code_Stop_MediaService`.
- `scripts/run_local.py`, `examples/README.md`, `.gitignore` 주석이 `sample_data` 를 참조한다 — 함께 정리.
- `.env.example` 에 CWD 기준임을 명시하고 `RCA_DATASET_ROOT` 를 추가.
- 리플레이어가 유일한 데이터 공급원이 되므로, 테스트는 fixture 파일 대신 리플레이어 출력이나 임시
  디렉터리를 쓴다.

## 미결

- **커밋 제외분(666MB)의 공유 위치.** `datasets/sn/README.md` 에 TODO 로 남아 있다.
- **`Normal_Baseline_*` 를 커밋 범위에 넣을지.** MVP 3종이 아니라 지금은 제외했으나, baseline 프로파일
  (ADR-002)을 원본에서 다시 생성해야 한다면 필요해진다. 현재는 `resources/baselines/sn_normal.json`
  이 커밋돼 있어 당장은 막히지 않는다.
- 리플레이어 배속(1x/5x/10x) 과 SDK 30초 루프의 상호작용 — 10x 재생 시 버퍼 윈도(ADR-001)가 실시간
  기준 210초를 그대로 쓸지, 재생 시간축으로 환산할지.
